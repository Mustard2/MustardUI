# Contributing

Thank you for contributing to this project!

This document outlines a few important practices to keep the project history clean and collaboration smooth.

---

## 📌 Commit History

- The `main` branch must always remain **linear**
- When merging a pull request, always use **Squash and Merge**
- Avoid merge commits on `main`

---

## 🐛 Bug Issues

- Bug issues should only be closed once a **fix has been released on `main`**
- Use the **Development section** in the issue to link related pull requests
- Ensure the issue is properly associated with the fix before closing

---

## 📚 Documentation Updates

- If a pull request requires documentation changes, please include the changes an administrator should make to the 
documentation
- A PR will only be merged once documentation is updated and aligned with the changes (if applicable)
- This ensures documentation always reflects the current state of the project

---

## 🚫 Content Guidelines

- Please do **not post NSFW Blender files, images, or videos** in Issues or Pull Requests
- If something sensitive or unclear needs to be shared, contact a maintainer first via:
  - Discord
  - Twitter / X

---

## 🤝 Final Notes

These guidelines may evolve over time as the project grows.  
Thank you for helping keep the project clean, consistent, and easy to maintain.
